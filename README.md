## 📖 О проекте

**Bunker36 Online** — это многопользовательская браузерная игра, где игроки оказываются перед лицом глобальной катастрофы и должны решить, кто достоин места в бункере для выживания человечества.

### 🎯 Концепция игры

Мир на грани уничтожения. Ядерная зима, восстание ИИ, зомби-вирус или вторжение инопланетян — катастрофа уже произошла. У вас есть бункер с ограниченным количеством мест. Каждый игрок получает уникального персонажа с профессией, состоянием здоровья, фобиями и багажом. 

**Задача:** Убедить других игроков, что именно вы заслуживаете места в бункере, и проголосовать за изгнание тех, кто менее полезен для выживания.

## 🎲 Игровой процесс

### Фазы игры

1. **Лобби** — Ожидание игроков (6-15 человек)
2. **Введение** — Знакомство с катастрофой и бункером
3. **Раунды** — Игроки по очереди раскрывают характеристики
4. **Дискуссия** — Обсуждение и аргументация
5. **Защита** — Игрок с наибольшим количеством голосов защищается
6. **Голосование** — Решение, кого изгнать
7. **Прощание** — Изгнанный игрок покидает бункер
8. **Финал** — AI-судья определяет судьбу выживших

### Карты действий

30 уникальных карт, которые могут:
- Изменить характеристики игроков
- Дать иммунитет или двойной голос
- Заблокировать раскрытие определенных карт
- Воскресить изгнанного игрока
- Связать судьбы двух игроков
- И многое другое...

### AI-судья

После завершения голосований, AI анализирует:
- Профессии и навыки выживших
- Состояние здоровья и возможность размножения
- Психологическую совместимость
- Соответствие условиям выживания катастрофы
- Ресурсы бункера

**Результат:** Вердикт о выживании или гибели бункера с детальным обоснованием.

## 🏗️ Архитектура

### Frontend

```
React 18 + TypeScript
├── Vite (сборка)
├── Tailwind CSS (стилизация)
├── Framer Motion (анимации)
├── Radix UI (компоненты)

### Backend

```
Supabase
├── PostgreSQL (база данных)
├── Realtime (WebSocket синхронизация)
├── Anonymous Auth (аутентификация)
└── RLS (Row Level Security)

### Синхронизация

- **Event-driven** архитектура через Supabase Realtime
- **Приоритетная очередь** для критичных событий
- **Батчинг** обновлений (100ms) для оптимизации
- **Оптимистичные обновления** для мгновенного UI
- **Fallback механизмы** при потере соединения

## 🔧 Ключевые компоненты

### GameContext

Центральный контекст управления состоянием игры:
- Синхронизация с Supabase
- Управление фазами
- Обработка действий игроков
- Таймеры и автопереходы

### useGameQuery

React Query хук для оптимизированной загрузки данных:
- Адаптивный polling (300ms в игре, отключен в лобби)
- Кеширование
- Автоматическая инвалидация

### useActionCards

Управление картами действий:
- Активация карт
- Выбор целей
- Отмена действий
- Применение эффектов

### AI Judge (useGameJudge)

Интеграция с DeepSeek AI:
- Анализ состава группы
- Оценка шансов выживания
- Генерация вердикта
- Fallback механизмы
